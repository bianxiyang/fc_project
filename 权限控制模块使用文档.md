# 权限控制模块使用文档

## 1. 模块概述

权限控制模块是基于Spring Security实现的角色和权限管理系统，用于控制用户对系统功能的访问权限。该模块通过`user_permissions`表实现用户和权限之间的多对多关联，支持细粒度的权限控制。

## 2. 核心组件

### 2.1 数据模型

- **Permission**: 权限实体，存储系统中的权限定义
- **UserPermission**: 用户权限关联实体，存储用户和权限的关联关系
- **FcUser**: 用户实体，存储用户信息

### 2.2 服务层

- **PermissionService**: 权限管理服务，提供权限的CRUD操作
- **UserPermissionService**: 用户权限管理服务，提供用户权限的分配和管理
- **FcUserService**: 用户管理服务，提供用户信息和权限查询

### 2.3 控制器

- **PermissionController**: 权限API控制器，提供权限相关的REST接口
- **PermissionPageController**: 权限管理页面控制器，提供权限配置页面

## 3. 权限配置流程

### 3.1 访问权限配置页面

1. 登录系统（需要ROOT角色）
2. 导航到 `/permission-config` 页面

### 3.2 配置用户权限

1. 在用户选择下拉框中选择要配置权限的用户
2. 系统会自动加载该用户当前的权限配置
3. 勾选或取消勾选权限复选框，配置用户的权限
4. 点击"保存权限配置"按钮保存配置

### 3.3 权限管理功能

- **全选/取消全选**: 快速选择或取消所有权限
- **权限搜索**: 通过关键词搜索权限，方便快速定位

## 4. 权限验证机制

### 4.1 登录时权限加载

当用户登录时，系统会：
1. 加载用户的角色权限（如ROLE_USER、ROLE_ADMIN）
2. 从`user_permissions`表加载用户的具体权限
3. 将这些权限添加到Spring Security的认证对象中

### 4.2 权限验证方式

1. **URL级别验证**: 通过SecurityConfig中的`authorizeHttpRequests`配置URL访问权限
2. **方法级别验证**: 使用`@PreAuthorize`注解验证方法访问权限
3. **代码级别验证**: 使用`UserPermissionService.hasPermission`方法验证用户权限

## 6. 开发指南

### 6.1 添加新权限

1. 在`InitPermissions`类中添加新的权限定义
2. 重启系统，权限会自动初始化到数据库

### 6.2 权限验证示例

#### URL级别验证

```java
// 在SecurityConfig中配置
.antMatchers("/api/matches/**").hasAnyAuthority("match_manage", "match_list")
```

#### 方法级别验证

```java
@PreAuthorize("hasAuthority('match_edit')")
public ResponseEntity<ApiResponse> updateMatch(Integer id, Match match) {
    // 方法实现
}
```

#### 代码级别验证

```java
if (userPermissionService.hasPermissionCode(userId, "match_score")) {
    // 执行比分录入操作
} else {
    // 无权限提示
}
```

## 7. 注意事项

1. **权限配置需要ROOT角色**：只有超级管理员可以配置用户权限
2. **权限继承关系**：权限按模块分组，勾选父权限不会自动勾选子权限，需要单独勾选
3. **权限生效时机**：权限配置保存后，用户需要重新登录才能生效
4. **默认权限**：新用户默认没有任何具体权限，只有角色权限
5. **权限清理**：删除用户时，系统会自动清理该用户的权限关联

## 8. 故障排查

### 8.1 权限配置不生效

- 检查用户是否重新登录
- 检查权限是否正确保存到数据库
- 检查权限代码是否与验证代码一致

### 8.2 权限页面加载失败

- 检查服务器是否正常运行
- 检查数据库连接是否正常
- 检查浏览器控制台是否有JavaScript错误

### 8.3 权限验证失败

- 检查用户是否拥有对应的权限
- 检查权限代码是否正确
- 检查权限验证逻辑是否正确

## 9. 性能优化

1. **权限缓存**：考虑添加权限缓存，减少数据库查询
2. **批量操作**：使用批量保存和删除操作，提高权限配置效率
3. **索引优化**：在`user_permissions`表的`user_id`和`permission_id`字段上添加索引
